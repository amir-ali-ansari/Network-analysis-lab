# Lab 01: Wireshark Traffic Analysis

## Objective
The objective of this lab was to analyze network traffic using Wireshark and understand how packets move across a network.

## Tools Used
- Wireshark
- TCP/IP knowledge

## Steps Performed
1. Opened a packet capture file in Wireshark.
2. Reviewed source and destination IP addresses.
3. Filtered traffic by protocol.
4. Identified TCP, DNS, and HTTP packets.
5. Looked for unusual traffic patterns.

## Findings
- Identified multiple communication sessions between hosts.
- Observed DNS requests used to resolve domain names.
- Reviewed TCP handshake behavior.
- Practiced reading packet details.

## What I Learned
This lab helped me understand how network traffic appears at the packet level. I learned how to use Wireshark filters, inspect protocols, and connect classroom networking concepts to real cybersecurity investigation work.

## Real-World Connection
SOC analysts and cybersecurity professionals use packet analysis to investigate alerts, detect suspicious traffic, and understand how systems communicate during security incidents.
